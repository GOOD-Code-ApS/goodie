# @goodie-ts/hono

HTTP controller decorators and Hono integration for goodie-ts. Provides route metadata decorators, a transformer plugin for compile-time route wiring, and runtime server infrastructure.

## Key Files

| File | Role |
|------|------|
| `src/metadata.ts` | `HONO_META` symbol keys, `ControllerMetadata`, `RouteMetadata` types |
| `src/controller.ts` | `@Controller(basePath?)` тАФ marks class as HTTP controller, stores basePath |
| `src/route.ts` | `@Get`, `@Post`, `@Put`, `@Delete`, `@Patch` тАФ method decorators via `createRouteDecorator()` |
| `src/validate.ts` | `@Validate({ json: schema })` тАФ attaches validation metadata for zod-validator middleware |
| `src/server-config.ts` | `ServerConfig` тАФ `@ConfigurationProperties('server')` bean with `host` and `port` defaults |
| `src/embedded-server.ts` | `EmbeddedServer` тАФ `@Singleton` that wraps `@hono/node-server`, uses `ServerConfig` for defaults |
| `src/plugin.ts` | Transformer plugin тАФ generates `createRouter()` and `startServer()` from controller metadata |
| `src/index.ts` | Public exports |

## HONO_META Keys (metadata.ts)

All are Symbols under the `HONO_META` object:
- `CONTROLLER` тАФ `ControllerMetadata` (`{ basePath }`)
- `ROUTES` тАФ `RouteMetadata[]` (`{ method, path, methodName }`)

## Transformer Plugin (`src/plugin.ts`)

The hono plugin is auto-discovered at build time via `"goodie": { "plugin": "dist/plugin.js" }` in package.json. It handles all controller-related scanning and code generation:

- **`visitClass`** тАФ detects `@Controller(basePath)`, calls `ctx.registerBean({ scope: 'singleton' })` to register the class as a bean, and initializes controller metadata
- **`visitMethod`** тАФ detects `@Get`/`@Post`/`@Put`/`@Delete`/`@Patch` route decorators and `@Validate` on methods, accumulates route definitions
- **`codegen`** тАФ reads `bean.metadata.controller` and generates per-controller route factories, `createRouter(ctx)`, `startServer(options?)`, per-controller `XxxRoutes` types + `createXxxClient()` factories, and the full `AppType` + `createClient()`

The transformer core knows nothing about HTTP methods, routes, or controllers. The hono plugin registers `@Controller` classes as singleton beans via `ctx.registerBean({ scope: 'singleton' })` in `visitClass`. All route scanning and codegen is fully owned by this package тАФ the DI core has zero HTTP knowledge.

## ServerConfig + EmbeddedServer

`ServerConfig` is a library bean shipped via `beans.json`. It uses `@ConfigurationProperties('server')` to bind from JSON config files or env vars:

```json
{ "server": { "host": "localhost", "port": 3000 } }
```

`EmbeddedServer` is a `@Singleton` that takes `ServerConfig` as a constructor dependency. The generated `startServer()` calls `EmbeddedServer.listen(router, options)` тАФ options override config defaults.

## Design Decisions

- **`@Controller` registers as singleton via plugin** тАФ the hono plugin calls `ctx.registerBean({ scope: 'singleton' })`, so `@Controller` alone is sufficient for bean registration
- **RPC support** тАФ each controller gets its own route factory function, exported `XxxRoutes` type, and `createXxxClient(baseUrl)` factory. `createRouter` composes them via `.route()`. Full `AppType` and `createClient(baseUrl)` are also exported. Per-controller splitting follows Hono's recommended pattern for larger applications
- **Methods receive Hono `Context` directly** тАФ no parameter decorator magic (Stage 3 has no param decorators)
- **Return value conventions**: `Response` passthrough, `undefined`/`null` returns 204, everything else is `c.json(result)`
- **Plugin-based codegen** тАФ route wiring is generated by this package's plugin, not by the transformer core
- **Controller metadata on beans** тАФ the hono plugin stores controller route data on `bean.metadata.controller` via `visitClass`/`visitMethod` hooks. The transformer core has zero knowledge of routes or HTTP methods.

## Testing

- **Transformer tests** (`packages/transformer/__tests__/controller-pipeline.test.ts`) тАФ verify `@Controller` registers as a singleton bean (the only thing the transformer core does)
- **Hono tests** (`packages/hono/__tests__/hono-plugin.test.ts`) тАФ verify the full plugin pipeline: scanning `@Controller`/`@Get`/`@Post` via `visitClass`/`visitMethod`, metadata population, and `createRouter`/`startServer` codegen
- No cross-package test dependencies

## Gotchas

- Route decorators are matched by name only (no import source verification), but only scanned on `@Controller` classes
- Controller variable names in generated code use collision-safe naming (`className:importPath` keying)
- `@Validate` generates `zValidator()` middleware тАФ requires `@hono/zod-validator` and `zod` as peer deps
